Install

Two packages, same app. Pick the one that matches how you manage software.

.deb Debian / Ubuntu

Integrates with apt; installs the desktop entry and icons system-wide.

  1. Download the package and its signature: 
    curl -LO https://github.com/Super-Submarine/vmforge/releases/latest/download/vmforge_amd64.deb
  2. Verify (see below), then install: 
    sudo apt install ./vmforge_amd64.deb
  3. Launch from your app grid, or run vmforge.

.AppImage Any distro

Single portable file. No root, no install — runs on any modern Linux.

  1. Download: 
    curl -LO https://github.com/Super-Submarine/vmforge/releases/latest/download/VMForge-x86_64.AppImage
  2. Verify (see below), then make it executable: 
    chmod +x VMForge-x86_64.AppImage
  3. Run it: 
    ./VMForge-x86_64.AppImage

Verify your download

Every wave-1 artifact ships with a detached GPG signature and a signed SHA256SUMS file. Verification takes ~30 seconds.

🔒 Check the signature before installing

Import the VMForge release key, then verify the checksum file and your artifact:

# 1. Import the release public key (also in the repo: keys/vmforge-release.asc)
curl -L https://github.com/Super-Submarine/vmforge/releases/latest/download/vmforge-release.asc | gpg --import

# 2. Verify the signed checksum manifest
curl -LO https://github.com/Super-Submarine/vmforge/releases/latest/download/SHA256SUMS
curl -LO https://github.com/Super-Submarine/vmforge/releases/latest/download/SHA256SUMS.asc
gpg --verify SHA256SUMS.asc SHA256SUMS

# 3. Check your artifact against the manifest
sha256sum -c SHA256SUMS --ignore-missing

Expected: Good signature from "VMForge Release Signing". Key fingerprint: [FINGERPRINT — filled in when VMFORGE_RELEASE_GPG_PRIVATE_KEY is provisioned]. If verification fails, do not install — report it.
